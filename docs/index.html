Prototype / Hackathon V1

Pre-trade execution quality, without trading.

OutcomeRail evaluates public Polymarket orderbook liquidity for a requested outcome and size, then returns a deterministic, independently verifiable receipt.

View architecture Read local API contract View source on GitHub

Public data only

Resolves a public Gamma market and reads its public CLOB orderbook snapshot. No account, credential, private feed, or trading endpoint is used.

Conservative verdict

Requested-size VWAP and visible depth produce PROCEED, REDUCE, or BLOCK. Freshness, spread, and price-gap policy can only tighten the result.

Verifiable receipt

A canonical input manifest, policy hash, source snapshot metadata, analysis result, and receipt hash are bound into one tamper-evident artifact.

Safety boundary: OutcomeRail V1 does not send orders, custody funds, connect wallets, settle payments, or make profitability claims.
